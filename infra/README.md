## Local database

Start Postgres 16 with `docker compose up -d db` from the repo root ([compose.yaml](../compose.yaml)). The API reads only `DATABASE_URL` — point it at the compose service locally or at a managed URL in cloud (`?sslmode=require`).

## Subdomain dev (nginx — local only)

Production uses one hostname per surface. Local dev mirrors that via nginx on **`<local-base>`** subdomains (from [`product.ts`](../packages/env/src/lib/product.ts); default `userepro.test`).

| Host pattern              | App                          | Upstream         |
| ------------------------- | ---------------------------- | ---------------- |
| `api.<local-base>`        | API (`apps/api`)             | `127.0.0.1:5000` |
| `console.<local-base>`    | Console (`apps/console`)     | `127.0.0.1:5001` |
| `docs.<local-base>`       | Docs (`apps/docs`)           | `127.0.0.1:5009` |
| `<local-base>` (apex)     | Marketing (`apps/marketing`) | `127.0.0.1:5002` |

See [hosts.example](./hosts.example) for the current generated list. `www.<local-base>` → 301 → apex.

**Config:** `infra/nginx/local.conf` (generated by `bun run sync:local-infra`). **Rename product:** `bun run configure-product <slug>`.

Cloud (Railway/Coolify) uses platform routing on long-running services — not this nginx file.

**Browser extension:** WXT dev uses `localhost:5555` directly — no nginx vhost.

### WSL2 + Windows browser

Typical setup: **nginx and dev servers in WSL2**, **browser on Windows**, **Cursor in WSL2**.

| Where        | Hosts file                              | Needed?                                                                |
| ------------ | --------------------------------------- | ---------------------------------------------------------------------- |
| WSL2 (Linux) | `/etc/hosts`                            | Yes — for `curl`, Playwright MCP, and tools running in WSL             |
| Windows      | `C:\Windows\System32\drivers\etc\hosts` | **Yes** — Windows Chrome/Edge/Firefox do **not** read WSL `/etc/hosts` |

Copy the same lines from [hosts.example](./hosts.example) into **both** files.

**Target IP:** usually `127.0.0.1` when WSL uses **mirrored networking** (Windows 11 22H2+). If the console URL fails from Windows but works inside WSL, point Windows hosts at the WSL IP instead:

```bash
hostname -I | awk '{print $1}'
```

Then use that IP in the Windows hosts file (WSL IP can change after reboot in legacy NAT mode).

**nginx** runs in WSL and listens on WSL loopback; mirrored mode forwards Windows `127.0.0.1:443` to WSL. Reload after conf changes:

```bash
sudo nginx -t && sudo systemctl reload nginx
```

**mkcert (Windows browser):** import WSL `rootCA.pem` into Windows Trusted Root via `certmgr.msc`. From WSL, run `wslpath -w "$(mkcert -CAROOT)/rootCA.pem"` for the File Explorer path — see [certs/README.md](./certs/README.md#windows-browser-wsl2).

### Setup

1. **Hosts** — merge [hosts.example](./hosts.example) into WSL `/etc/hosts` **and** Windows hosts (see [WSL2 + Windows](#wsl2--windows-browser)).
2. **TLS + nginx** — `bun run setup:local` from repo root (or manual steps in [certs/README.md](./certs/README.md)).
3. **Windows CA trust** — `certmgr.msc` → Trusted Root → import `rootCA.pem` ([certs/README.md#windows-browser-wsl2](./certs/README.md#windows-browser-wsl2)).
4. **Env** — `cp apps/api/.env.example apps/api/.env` and set `BETTER_AUTH_SECRET`.
5. **Run** — `bun run dev` → https://console.userepro.test (or your `product.ts` local base)
